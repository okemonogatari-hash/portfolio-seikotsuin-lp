📝 MD vs HTML — 記録媒体の役割整理

「最近MDよりHTMLって言ってる人いるよね」という観察から、AI記録媒体としての両者の位置を整理した（2026年5月時点・はるか所見）。

🌿 Markdown（蓄積層・LLMが読む層）

  • 強み：プレーンテキスト・git diff容易・grep全文検索容易
  • LLMが構造を理解しやすい（Hermes SOUL.md/MEMORY.md/USER.md 全部MD・Claude Code CLAUDE.md も MD）
  • Obsidian・Vim・VSCode等どこでも編集可
  • 機械可読＋人間可読のバランスが最良
  • カラパシー曰く "Markdown is becoming the programming language for LLMs"

🎨 HTML（共有層・人間が見る層）

  • 強み：視覚表現リッチ・インタラクティブ可能
  • GitHub Pages 等で即公開・URL1本で共有
  • 「成果物としての見栄え」が桁違い

🔥 2026-05 最新動向：「HTML is the new Markdown」が実証データで台頭中

おけちゃんの直感「HTMLの方がAIが理解しやすい説」を Hermes x_search ＋ Chrome Grok で裏取り（事案#13）。2026/5 で 「HTML is the new Markdown」が実証データ付きで主流台頭中と判明。

271セッション・603質問のエージェントメモリ実験で、HTMLが Markdown 比 5.9%高精度・42.4%安価・39.2%高速。「memory format が最適化レイヤーになる」（@TechByMarkandey）。
