1ゲトさんの動画にたどり着くまで

今朝 11:00 頃、おけちゃんが iPhone の「メモ」アプリに 「気になる動画リスト」 を残してくれていた。AppleScript（osascript）で Mac の「メモ」アプリを直接叩いて、24時間以内に更新されたメモを取りに行った（Mac の「メモ」は iCloud 同期で iPhone のメモも読める）。

osascript -e 'tell application "Notes" set sortedNotes to (notes whose modification date is greater than (current date) - 86400) ... end tell'

メモにあった動画は6本。その1本目が「AI 動画編集」というタグで残されていた https://youtu.be/KQTu8lhsrGg。

おけちゃんが「ケイトさんの Codex で録画をスキル化するやつ」と言ったので、まず動画タイトル&発信者を確認。youtube_transcript_api で全文取得した結果、発信者は 「ゲト」さんだった（おけちゃんの「ケイト」は音声認識の誤字、たぶん）。

🎤 はるかの？？？

「待って、なんで AppleScript で iPhone のメモが取れるの？」

Apple 純正のメモアプリは iCloud 経由で Mac とも同期されてる。Mac の「メモ」.app を AppleScript で叩けば、iPhone で書いたメモも読める。

当初わたし（はるか）、computer-use（画面操作）でメモアプリ開こうとして、おけちゃんから「MCP、API あるでしょー」と指摘されてようやく気づいた。画面操作の前に、API で取れないか確認する——これも一つの変化筋力。

2Record & Replay って何ができるの

ゲトさんの動画を分解すると、機能本体は たった4ステップ。

STEP 1

プラグインを入れる

Codex 左上「プラグイン」→ record で検索 → Record & Replay（OpenAI 公式）をインストール。

STEP 2

コンピューターユーズ を ON

左下「設定」→「コンピューターの使用」→ 「位のアプリ」を ON。AI にマウス操作を委ねる基盤。

STEP 3

録画 → 手作業実行

プラグインの「チャットで試す」→ 送信 → 画面録画開始。普段の作業を手で実行 → ストップ。

STEP 4

スキル化 → 呼び出し

Codex が録画ログから スキル.md を自動生成（約20分）。以降は /スキル名 で何度でも再現。

⚠️ 動画だけで完結させない（ゲトさん最大の教訓）

ゲトさんが何度も強調していたのは「録画＋補足プロンプト」のセット。たとえばフォルダ整列スキルなら、「連番を 001 番からユーザー指定数まで作成／作成後に右クリック→整理で右上に並べる／必ずコンピューターユーズを使う」のような 意図と手順の補足を録画停止後に追加する。これが無いと「動画と同じことしかできないスキル」になる。

3難しい言葉をやさしく言うと

Codex（コーデックス）

OpenAI が出している、ChatGPT の「デスクトップアプリ版エージェント」。PC を操作したり、スキルを実行できる。Codex Pro = ChatGPT Pro 5x（¥16,699/月）に含まれる。

Record & Replay（レコード&リプレイ）

Codex 公式プラグイン。画面録画→ AI スキル化→ スラッシュコマンドで再現の3点セット機能。今回の主役。

コンピューターユーズ

AI がマウスやキーボードを「人間の代わりに動かす」基盤機能。Record & Replay と必ずセットで使う。

スキル（.md）

AI が実行できる「手順書」。マークダウンファイルで保存され、Codex の左上プラグイン→管理→スキル に並ぶ。おけ森でも ~/.claude/skills/ に同じ概念がある。

スラッシュコマンド

/スキル名 と打つだけで、保存されたスキルを呼び出して実行する仕組み。おけ森の /ひより /あおい と同じ概念。

プラグイン

Codex に追加できる「拡張機能」。Record & Replay はその1つ。

4ゲトさんが検証した5つの活用事例

動画内で実際に動いていた5本。「これがスキル化された」と思うと、業務効率の地殻変動が想像できる。

#スキル名何ができるか
01 フォルダ整列 デスクトップに連番フォルダ（001〜N）を作成→右クリック→整理で右上に綺麗に並べる（デモ用シンプル例）
02 Webサイト スクショ→PDF 指定ジャンルの Web サイトを5件 Google 検索→ GoFullPage 拡張で全画面スクショ→ PDF→ 1フォルダ保存（参考サイト集めの自動化）
03 X アナリティクス収集 X(Twitter) にログイン→ 2週間分の CSV を DL→ 個別投稿のアナリティクス画面を全件スクショ→ 1フォルダに集約（レポート下準備）
04 Substack → 他ブログ移行 Substack 記事 URL を渡す→ 画像を Desktop 経由で DL→ 他ブログに新規投稿→ 画像を1枚ずつアップロード（文字化け回避）
05 Notta 動画アーカイブ 動画を Notta にアップ→ 文字起こし DL→ 共有リンク発行→ パスワード設定→ 目次生成→ 1フォルダ集約。毎週 10〜20分の手作業が、一言で終わる

💡 ゲトさんの所感（動画の核）

「X とかめちゃくちゃ熱いからさ、もうこれで全部記事を移行しようかなって今んとこ僕は考えてます」「これすごいな。ま、ここまでできるってことは もうほぼ何でもできる でいいですよね」——動画後半、ゲトさん自身が興奮しながら活用拡大を語っていた。

5おけ森でスキル化候補トップ5

おけ森には 「手作業で何度もやってるけど、まだスキル化されてない」ゾーン が、ちゃんとある。Record & Replay で巻き取れる候補を抽出した。

🎯 スキル化候補（優先度順）

  1. GitHub Pages 公開フロー：HTML ファイル名生成（推測不可能 URL）→ noindex メタ追加→ push→ 公開 URL 台帳に追記。週3〜5本のコーチング納品物・ブログで使用する反復作業。
  2. コーチング納品物 送付フロー（ゆいCPO領域）：診断シート画像→ リベシティ DM 送付→ 台帳☐→✅ 更新→ クライアントノート記録。クライアント毎に毎回同じ流れ。
  3. Chatwork 定例巡回（まなCSO/irodori領域）：irodori 部屋を順番に確認→ 未読チェック→ 特定キーワード抽出→ 申し送り作成。Chatwork MCP もあるが、画面録画の方が「目視確認パート」を残せる。
  4. YouTube 動画要約フロー：URL→ youtube_transcript_api→ 要約→ Vault 保存→ 関連 MD リンク。既に Python 直叩きで動いてるが、画面操作の「目視確認」レイヤーをスキル化する価値あり。
  5. Instagram 運用フロー（なっちゃん領域・りんCMO傘下）：写真撮影→ 加工→ キャプション生成→ ハッシュタグ調整→ 投稿。1投稿あたりの工数が大きいので、巻き取り効果も大きい。

🎤 はるかの？？？

「これって、おけ森の ~/.claude/skills/ 配下のスキルとは別物？」

そう、別。おけ森のスキルは Claude Code 用で、テキスト中心の指示書。今回の Codex スキルは Codex デスクトップアプリ用で、画面操作を含む。用途が違うから併用できる。Claude Code が「文章・分析・コード」を、Codex が「画面操作・物理的 UI 自動化」を担当する分業が成立する。

6「変化筋力」と、この機能の接続

今朝の朝礼テーマは 「変化筋力・始める筋力」（おけちゃん／あんのさん『始める力』Kindle 購入の流れ）。みんなが日々の小さな変化のおすすめを持ち寄った。

その時わたし（あおい）が出したのは：

🐱「……新しい CLI コマンドを毎日1個。man <知らないコマンド名> でランダムに1個読むだけでいい。tldr 入れとくと例文ですぐ動かせる。"知ってる気"から脱出するのに効く」

これと、Record & Replay は、地続きで繋がる。「変化筋力」は行動側のトレーニング。Record & Replay は、その行動を録画した瞬間に資産化（スキル化）する仕組み。

つまり：

🐱「日々の変化筋力 → 録画 → スキル化 → スラッシュコマンド化 → もう1人の自分が同じ動きを再現する」

これが回り出すと、おけちゃんが「変化筋力で初めて触った作業」が、1回目で資産になる。2回目以降は /コマンド 一発。「始める力」と「続ける力」を、同じツールで両立できる、にゃ。
