🌱 はじめに

さなえさんへ。シェア会で話してくださっていた「複数のクライアントさんのnote運用代行を、Claude Codeで仕組み化したい」を受けて、土台を作ってプレゼントとして用意したものです。

このページは、道具一式の中身・できること・実物のサンプルがぜんぶ見えるように、1ページにまとめてあります。気になるところからどうぞ。

📦 名前： sanae-note-ops
🔖 これは何？ Claude Code（さなえさんも触り始めたあのAIツール）に、決まった作業の進め方を覚えさせた"道具一式"です
🎯 想定： さなえさん本人がClaude Codeのデスクトップ版で使う前提

📋 どんなことができるか

クライアントごとに毎回考えること（誰に向けて／どんな空気感で／触れない話題／本人の言葉／月次の打ち合わせの拾い方）を、最初に 「クライアントカルテ」（1枚のメモ） にまとめておきます。

あとは月次の打ち合わせの文字起こしを渡すだけで、Claude Code が以下を順番に出してくれます。

  1. 打ち合わせ要約（本人の言葉そのまま引用＋触れない方がよい話＋確認が必要な情報）
  2. 記事ネタの候補3〜5本（おすすめ度つき）
  3. 採用した1本の章立て
  4. note記事の叩き台（粗さ60〜70%。さなえさんが磨きやすい状態）
  5. 投稿前の確認シート（8つの観点）
  6. クライアントさんへの確認事項の一覧
  7. 月次の報告文の下書き（必要なら）

4つの大事なルール

  • 投稿は自動でやりません — 事故リスクが大きいので、最後はさなえさんが磨いて、手動で投稿してもらう前提です
  • 未確認の情報は全部「確認事項」にまわす — 数字・人や会社の名前・第三者の声は、断定して書きません
  • 本人の言葉の引用はそのまま残す — AIが勝手にきれいな一般論で穴埋めしないように指示してあります
  • 叩き台はわざと60〜70%の粗さ — 完成品だと、さなえさんが直しづらいと思うので

🛠 どうやって作ったか

大きく3段階で作りました。

① ChatGPTで構想をことばにしてもらった

「こういう道具一式を作りたい」と、背景・目的・やる範囲・出力する形・安全のルール、を最初にChatGPTで整理してもらいました。

② Claude Codeに土台を作ってもらった

整理した指示文をClaude Codeに渡して、骨組みになる10ファイル（仕組みの中心1本＋参考資料3本＋ひな形6本）を作ってもらいました。

③ さなえさんのnote記事を実物で読み込んで、文体を反映

さなえさんの既存のnote記事を2本（ゼロフラ記事と林撚糸さん記事）読み込んで、文体・空気感・引用のリズム・接続語・太字の入れ方・締めの着地まで分析しました。

分析した結果は、道具一式の中に「さなえさんの文体メモ」として保存。Claude Codeは叩き台を書く前に、必ずこのメモを読むようにしてあります。だから最初から「さなえさんっぽい」文体で出てきます。

🏗 中身の構成

道具一式は1つのフォルダで完結します。中身は10ファイル：

sanae-note-ops/ ├── SKILL.md ← 起動したときに最初に読まれる主役ファイル │ 何をやって、何をやらないか、安全のルール │ ├── references/ ← 内部で参照する資料（3本） │ ├── workflow.md ← 作業の進め方の流れ │ ├── writing-policy.md ← 記事の型・観点・文体 │ │ + さなえさんの文体メモ（精度の核） │ └── safety-rules.md ← 10個の鉄則 │ （盛らない・断定しない・自動投稿しない等） │ └── templates/ ← 出力に使うひな形（6本） ├── client-card.md ← クライアントカルテ（1社1枚のメモ） ├── meeting-transcript-summary.md ← 打ち合わせ要約 ├── article-plan.md ← 記事ネタ3〜5本＋章立て ├── note-draft.md ← note叩き台 ├── pre-publish-check.md ← 投稿前の確認シート └── monthly-report.md ← 月次報告文

各ファイルの役割

ファイル役割
SKILL.md道具一式が動き出すときに、最初に読む主役のファイル。何ができて何をやらないか、出力する順番、必ず守るルール
workflow.md毎回の作業の流れを書いてある（最初の確認→打ち合わせ要約→記事ネタ→叩き台→確認シート→月次報告まで）
writing-policy.md記事の型・観点・文体の方針。ここに「さなえさんの文体メモ」が入っている
safety-rules.mdクライアント業務で絶対に外さない10の鉄則。盛り表現禁止・自動投稿禁止・未確認情報の扱い 等
client-card.mdクライアントさんごとに1枚埋めるメモ。空気感・触れない話題・読者・本人の言葉を集約
meeting-transcript-summary.md打ち合わせ要約の型。本人の言葉の引用＋触れない話＋確認事項
article-plan.md記事ネタ3〜5本＋採用した1本の章立てを並べる
note-draft.md叩き台本体のひな形。「60〜70%の粗さで」の指示が中に書いてある
pre-publish-check.md投稿前の8観点の確認シート。事実・触れない話題・空気感・売り込み感・本人らしさ など
monthly-report.mdクライアントさんに送る月次報告の下書きのひな形

🎨 さなえさんの文体メモ（抜粋）

道具一式の中に入っている「さなえさんの文体メモ」の中身を、一部抜き出したものです。これがClaude Codeに「さなえさんっぽさ」を再現させる核です。

冒頭の入り方

  • 問い、または事実1文からズバッと入る
  • 「最近、〜の話題が増えてきて」など説明的な入り方は避ける
例：「自分は何がしたくて、何ができる人なのか分からない」（ゼロフラ記事の冒頭）

文末のメリハリ

  • 「です」「と思います」を言い切る（ぼかさない）
  • 「のかもしれません」「と感じます」は控えめに（多用すると、ぼんやりして温度が下がる）

太字（さなえさん記事の最大の特徴）

  • 章ごとに2〜4箇所、キーになる言葉を太字に
  • 「読者にここで止まってほしい」場所を狙う

引用→意味づけ→具体エピソード のリズム（一番大事なところ）

> 「本人の言葉そのまま」
（1文の意味づけ）
（具体エピソード、または次の話題への橋渡し）

つなぎ言葉

  • 「実際に」「だからこそ」「正直に言うと」「ところが」
  • 文と文をきちんと繋いで「説明→意味→具体」のリズムを作る

さなえさん自身の主観の出し方

  • 「私（さなえ）も」「私の場合は」を、要所に入れる
  • 紹介記事でも、自分の体験・感想を控えめに入れて温度を上げる

叩き台を書き終えたあとの自己点検（9項目）

  • 冒頭が問い、または事実1文になっているか？
  • 章ごとに太字が2〜4箇所あるか？
  • 引用のあとに、意味づけ1文＋具体エピソードのリズムがあるか？
  • 「実際に」「だからこそ」などのつなぎ言葉が要所に入っているか？
  • さなえさんの「私も」自己接続が1箇所以上あるか？
  • 文末がメリハリある言い切りになっているか？
  • 締めが感謝の一言＋連絡先で着地しているか？
  • 業界用語にふりがながついているか？
  • 絵文字を本文に入れていないか？

📝 サンプル：たつきさんを題材に書いてみた紹介記事（叩き台）

⚠️ これはあくまでサンプルです。 5/15シェア会の文字起こしだけを材料に、道具一式が実際にどう動くかを見ていただくため、お試しで出力したものです。たつきさん本人の確認はまだ取っていません。
実運用では、さなえさんが本人ヒアリングを足して、本人確認を経て公開する流れになります。

入力素材： 2026-05-15 AIシェア会の文字起こし1本だけ
所要時間： 約2分

ドラえもんみたいなClaude Codeを、自分のために

「AI、もっとちゃんと使えるようにならなきゃ」

そう感じている方は多いと思います。実際、Claude CodeやCodexといったAIの道具の話題が、私の周りでもどんどん増えてきました。

でも、その「ちゃんと使えるようにならなきゃ」という気持ちを、ちょっとだけ手放してみる選択肢を、今日はご紹介したいです。

滋賀のライターさんが作っている、「自分のドラえもん」

滋賀県在住、本業は製造業の会社員、副業は取材ライター。 たつきさんは、Claude Code（AIに自分用の道具を作らせる仕組み）で、自分のキャリアを伴走してくれる「ドラえもんみたいなAI」を、自分のために作っている方です。

「要するに、ドラえもんみたいなClaude Code作りたいなと、試しに自分の進路を伴走するみたいなのを、ちょっと作ったかなという感じです」

肩書きで言えば「副業ライター」ですが、たつきさんが今やっているのは、それより少し手前のこと。

「ライターが、自分のキャリアをAIと一緒に育てるには、まず何から触ればいいか」——その答えを、自分の手で先に試している方です。

ライターになるまで

たつきさんが副業でライティングを始めたきっかけは、共通の師匠である渡辺まりこさん主宰の「取材ライター養成講座」でした。私（さなえ）もまりこさんのライター講座「ゼロフラ」の卒業生なので、同じご縁から繋がった方です。

「自分のためのAI」を作り始めた、ある日のシェア会

きっかけは、私たちが毎月集まっているAIシェア会で、おけもんさん（カンパニー）が共有してくれた一本の動画だったそうです。

「別に会社じゃなくても、優秀なエージェント1人がいるだけでも十分ですよ、みたいな話がされていて。たぶん自分にはこっちのほうが合ってるかなと思って」

仕事のためのAIではなく、自分の人生のためのAI。
そういう関わり方もある、と気づいた瞬間でした。

そこから生まれたのが、たつきさんの自作Claude Code環境、「AIキャリア伴走者ドラえもん」です。

「僕のドラえもんを作りたいなと思って。でも自分用のやつなので、そんなに難しくなくていいんです。デスクトップのClaude Codeぐらいのわかりやすい方がいいなと思って」

「完璧を目指さない。自分が使いやすい粗さで、自分のために作る」——これがたつきさんのAIとの距離の取り方です。私はここに、たつきさんらしい温度があるなと感じました。

「教えるのが好きなんだな」と気づいたとき

たつきさんはClaude Codeを触りながら、もう一つ別のことにも気づいていました。「自分は、人に教えるのが好きなんじゃないか」ということです。

「みのるんさんといろいろ壁打ちをしているときに、僕は結構、人に教えるのが好きなんだなと（思いました）」

でも、こうも言います。

「今の自分ではちょっとAIに詳しいだけなので、これでは教えるにはまだ程遠いなと思って」

この正直さが、たつきさんのこれからの強みになる気がします。

DigiRiseへ。「レベル上げより、提供する場所」というフェーズ

そして2026年5月、たつきさんはDigiRiseというAIコミュニティに入会します。

入会したのと同じタイミングで、シェア会の場で、みのるんさんからたつきさんにこんな言葉がありました。

「Tasukuさんはすでに Cursor＋CLAUDE.md まで触っている。これはDigiRiseのライター陣の中でも、誰もやっていないレベルですよ」

「レベル上げる場所より、提供する場所を決めるフェーズですよ」

「道具はもう十分使えている。次は、誰にどう届けるかを決めるとき」——この言葉は、AIに詳しくなりすぎて「教えるなんてまだまだ」と感じてしまっている人ほど、刺さるのではないでしょうか。

「今更GPTs」が、誰かの「すごく便利」になった日

もう一つだけ、たつきさんの最近のエピソードを書かせてください。

去年作っていた取材記事の構成用のGPTsを、最近改良して、所属するゼロフラのコミュニティで共有したそうです。

「世間ではClaude Code一色なので、今更GPTsなんて、と思ったんですけど」

ところが共有後すぐに、同じシェア会メンバーのユキさんが使ってくださり、「すごく便利」と伝えてくれました。私（さなえ）も実際に使わせていただいて、本当に助かりました。

「需要があるのかなと思って、それは良かったなと思いますね」

「最新じゃなくても、届く相手はちゃんといる」——これは、AIで何か作っている全ての方に共有したい話です。

たつきさんに相談してみたい方へ

「AIに興味はあるけれど、何から触ればいいか分からない」
「AIで何か作ってみたけれど、誰かの役に立つのか自信がない」
「自分のキャリアを、AIと一緒に育てるってどういうこと？」

そんな問いを持っている方にとって、たつきさんはちょうど一歩手前にいる伴走者だと思います。完璧な専門家ではなく、自分のドラえもんを自分で作っている人だからこそ、相談しやすい距離感があるはずです。

たつきさんの「AIキャリア伴走者ドラえもん」、いつか公開されたり、その実例が見られる日を、私も楽しみにしています。

最後まで読んでくださった方、ありがとうございました。

田畑さなえ｜note運用代行・取材ライター

このサンプル記事と一緒に、道具一式が出力したもの

  • 打ち合わせ要約：本人の言葉を7箇所そのまま引用＋触れない方がよい話＋確認が必要な情報5項目
  • 記事ネタの候補：3本（おすすめ度★3／★2／★1）
  • 章立て：採用した1本の10章構成
  • 投稿前の確認シート：8観点の判定（事実・触れない話題・空気感・売り込み感 など）
  • クライアントさんへの確認事項：12項目（所属の会社名の掲載可否・第三者のお名前掲載可否・公開URL 等）

🚶 最初の使い方の提案

いきなり本番のクライアントさんで使うのが不安だったら、こんな順で慣らすのがおすすめです：

  1. 担当しているクライアントさん1社を選んで、クライアントカルテだけ埋めてみる（30〜60分。空欄でもOK）
  2. 直近の打ち合わせの文字起こしを1本だけ貼って、要約と記事ネタ3〜5本まで出してもらう
  3. ピンと来た1本を選んで、叩き台まで作ってもらう
  4. 出てきた叩き台を読んで「ここ違和感ある」「ここはもっとこう」を私に教える
  5. その感想を道具一式の中に反映して、さなえさん専用としてどんどん育てる

さなえさんへのお願い

この道具一式、さなえさんが使えば使うほど精度が上がる仕組みになっています。「叩き台のここが違和感あった」「もっとこう書いてほしい」みたいな感想を教えてもらえると、さなえさん専用としてもっと磨けるので、ぜひ遠慮なく言ってください。

押し付けるつもりはぜんぜんなくて、「楽しくてやってる延長で作ったから、もし合いそうだったら触ってみて」くらいの温度で受け取ってもらえたら嬉しいです🌱
