🌱 はじめに

さなえさんへ。シェア会で話してくださっていた「複数クライアントのnote運用代行をClaude Codeで仕組み化したい」を受けて、骨組みを作ってプレゼントとして用意したものです。

このページは、スキルの中身・できること・実物のサンプルが全部見えるように1ページにまとめてあります。気になるところからどうぞ。

📦 名前： sanae-note-ops（Claude Code Skill）
🔖 バージョン： v1.1（さなえさん実物note記事のトーン分析を反映済み）
🎯 想定： さなえさん本人がClaude Codeのデスクトップアプリで使う前提

📋 このスキルでできること

クライアントごとに毎回考えること（誰に向けて／どんなトーンで／NGテーマ／本人語／月次MTGの拾い方）を クライアントカルテ にまとめておけば、あとは月次MTGの文字起こしを渡すだけで、Claude Codeが以下を順番に出してくれます。

  1. MTG要約（本人語そのまま引用＋触れない話＋要確認情報）
  2. 記事ネタ候補3〜5本（推奨度付き）
  3. 採用1本の章立て
  4. note叩き台（粗さ60〜70%でさなえさんが磨きやすい状態）
  5. 投稿前チェック（8カテゴリ）
  6. クライアントへの確認事項リスト
  7. 月次納品レポートの下書き（必要なら）

4つの大事なルール

  • 投稿は自動化していません — 事故リスク大なので最後はさなえさんが磨いて手動投稿の前提
  • 未確認情報は全部「確認事項」に回す — 数字・固有名詞・第三者の声は断定で書かない
  • 本人語の引用はそのまま残す — AIっぽい一般論で穴埋めしないように指示済み
  • 叩き台はわざと60〜70%の粗さ — 完成品だと直しづらいと思うので

🛠 どうやって作ったか

大きく3段階で作りました。

① ChatGPTで構想を言語化

「こういうスキルを作りたい」と背景・目的・スコープ・出力フォーマット・安全ルールをまずChatGPTに整理してもらいました。

② Claude Codeで骨組みを生成

整理した指示文をClaude Codeに渡して、SKILL.md・references（3本）・templates（6本）の合計10ファイルの骨組みを生成。

③ さなえさんのnote記事を実物で読み込んで、トーンを反映

さなえさんの既存note記事を2本（ゼロフラ記事と林撚糸記事）defuddleで取得して、文体・温度・引用のリズム・接続語・太字の入れ方・締めの着地まで分析。スキル内部の references/writing-policy.md に「さなえさん実物観察ノート」として残しました。

Claude Codeは叩き台を書く前に必ずこのノートを読むようにしてあるので、最初から「さなえさんっぽい」文体で出力されます。

🏗 スキルの中身（構成）

スキルは1フォルダで完結する設計です。中身は10ファイル：

sanae-note-ops/ ├── SKILL.md ← 起動時の振る舞い・出力順序・安全ルール │ ├── references/ ← 内部参照ドキュメント（3本） │ ├── workflow.md ← Step 0〜7の実行フロー │ ├── writing-policy.md ← 早苗さん記事5タイプの型・文体・観点 │ │ + 早苗さん実物観察ノート（v1.1の核） │ └── safety-rules.md ← 10の鉄則（未確認情報・盛り表現禁止 etc） │ └── templates/ ← 出力テンプレート（6本） ├── client-card.md ← クライアントカルテ（一言ポジション込み） ├── meeting-transcript-summary.md ← MTG要約（本人語引用＋要確認リスト） ├── article-plan.md ← 記事ネタ3〜5本＋採用1本の章立て ├── note-draft.md ← note叩き台（60〜70%粗さ） ├── pre-publish-check.md ← 投稿前チェック（8カテゴリ） └── monthly-report.md ← 月次納品レポートのテンプレ

🎨 早苗さんトーン観察ノート（抜粋）

スキル内部の references/writing-policy.md 末尾に入っている「早苗さん実物観察ノート」の抜粋です。これがClaude Codeに「早苗さんっぽさ」を再現させる核です。

冒頭の入り方

  • 問い or 事実1文からズバッと入る
  • 「最近、〜の話題が増えてきて」など説明的な導入は避ける
例：「自分は何がしたくて、何ができる人なのか分からない」（ゼロフラ記事冒頭）

文末のメリハリ

  • 「です」「と思います」を言い切る（ぼかさない）
  • 「のかもしれません」「と感じます」は控えめに（多用すると曖昧で温度が下がる）

太字（早苗さん記事の最大の特徴）

  • 章ごとに2〜4箇所 キーフレーズを太字化
  • 「読者にここで止まってほしい」場所を狙う

引用→解釈→具体 のリズム（核中の核）

> 「本人語そのまま」
（1文の解釈・意味付け）
（具体エピソード or 次の話題への橋渡し）

接続語

  • 「実際に」「だからこそ」「正直に言うと」「ところが」
  • 文と文をきちんと繋いで「説明→意味→具体」のリズムを作る

早苗さんの主観の出し方

  • 「私（さなえ）も」「私の場合は」を要所に入れる
  • 紹介記事でも自分の体験・感想を控えめに入れて温度を上げる

叩き台書き終わりのチェックリスト（9項目）

  • 冒頭が問い or 事実1文か？
  • 章ごとに太字が2〜4箇所あるか？
  • 引用後に1文の解釈→具体エピソードのリズムがあるか？
  • 「実際に」「だからこそ」など接続語が要所に入っているか？
  • 早苗さんの「私も」自己接続が1箇所以上あるか？
  • 文末がメリハリある言い切りになっているか？
  • 締めが感謝の一言＋連絡先で着地しているか？
  • 業界用語にふりがなが付いているか？
  • 絵文字を本文に入れていないか？

📝 デモ：たつきさんを題材に書いてみた紹介記事（叩き台 v2）

⚠️ これはあくまでサンプルです。 5/15シェア会のトランスクリプトだけを素材に、スキルが実際にどう動くかを見ていただくために試しに出力したものです。たつきさん本人の確認は取っていません。
実運用では、さなえさんが本人ヒアリングを足して、本人確認を経て公開する流れになります。

入力素材： 2026-05-15 AIシェア会のCirclebackトランスクリプト1本だけ
所要時間： 約2分

ドラえもんみたいなClaude Codeを、自分のために

「AI、もっとちゃんと使えるようにならなきゃ」

そう感じている方は多いと思います。実際、Claude CodeやCodexといったAI開発ツールの話題が、私の周りでもどんどん増えてきました。

でも、その「ちゃんと使えるようにならなきゃ」という気持ちを、ちょっとだけ手放してみる選択肢を、今日はご紹介したいです。

滋賀のライターさんが作っている、「自分のドラえもん」

滋賀県在住、本業は製造業の会社員、副業は取材ライター。 たつきさんは、AI開発ツール「Claude Code」で、自分のキャリアを伴走してくれる「ドラえもんみたいなAI」を自分のために作っている方です。

「要するに、ドラえもんみたいなClaude Code作りたいなと、試しに自分の進路を伴走するみたいなのを、ちょっと作ったかなという感じです」

肩書きで言えば「副業ライター」ですが、たつきさんが今やっているのは、それより少し手前のこと。

「ライターが、自分のキャリアをAIと一緒に育てるには、まず何から触ればいいか」——その答えを、自分の手で先に試している方です。

ライターになるまで

たつきさんが副業でライティングを始めたきっかけは、共通の師匠である渡辺まりこさん主宰の「取材ライター養成講座」でした。私（さなえ）もまりこさんのライター講座「ゼロフラ」の卒業生なので、同じご縁から繋がった方です。

「自分のためのAI」を作り始めた、ある日のシェア会

きっかけは、私たちが毎月集まっているAIシェア会で、おけもんさん（カンパニー）が共有した一本の動画だったそうです。

「別に会社じゃなくても、優秀なエージェント1人がいるだけでも十分ですよ、みたいな話がされていて。たぶん自分にはこっちのほうが合ってるかなと思って」

仕事のためのAIではなく、自分の人生のためのAI。
そういう関わり方もある、と気づいた瞬間でした。

そこから生まれたのが、たつきさんの自作Claude Code環境、「AIキャリア伴走者ドラえもん」です。

「僕のドラえもんを作りたいなと思って。でも自分用のやつなので、そんなに難しくなくていいんです。デスクトップのClaude Codeぐらいのわかりやすい方がいいなと思って」

「完璧を目指さない。自分が使いやすい粗さで、自分のために作る」——これがたつきさんのAIとの距離の取り方です。私はここに、たつきさんらしい温度があるなと感じました。

「教えるのが好きなんだな」と気づいたとき

たつきさんはClaude Codeを触りながら、もう一つ別のことにも気づいていました。「自分は、人に教えるのが好きなんじゃないか」ということです。

「みのるんさんといろいろ壁打ちをしているときに、僕は結構、人に教えるのが好きなんだなと（思いました）」

でも、こうも言います。

「今の自分ではちょっとAIに詳しいだけなので、これでは教えるにはまだ程遠いなと思って」

この正直さが、たつきさんのこれからの強みになる気がします。

DigiRiseへ。「レベル上げより、提供する場所」というフェーズ

そして2026年5月、たつきさんはDigiRiseというAIコミュニティに入会します。

入会したのと同じタイミングで、シェア会の場で、みのるんさんからたつきさんにこんな言葉がありました。

「Tasukuさんはすでに Cursor＋CLAUDE.md まで触っている。これはDigiRiseのライター陣の中でも、誰もやっていないレベルですよ」

「レベル上げる場所より、提供する場所を決めるフェーズですよ」

「ツールはもう十分使えている。次は、誰にどう届けるかを決めるとき」——この言葉は、AIに詳しくなりすぎて「教えるなんてまだまだ」と感じてしまっている人ほど、刺さるのではないでしょうか。

「今更GPTs」が、誰かの「すごく便利」になった日

もう一つだけ、たつきさんの最近のエピソードを書かせてください。

去年作っていた取材記事構成用のGPTsを、最近ブラッシュアップして、所属するゼロフラのコミュニティでシェアしたそうです。

「世間ではClaude Code一色なので、今更GPTsなんて、と思ったんですけど」

ところがシェア後すぐに、同じシェア会メンバーのユキさんが使ってくださり、「すごく便利」と伝えてくれました。私（さなえ）も実際に使わせていただいて、本当に助かりました。

「需要があるのかなと思って、それは良かったなと思いますね」

「最新じゃなくても、届く相手はちゃんといる」——これは、AIで何か作っている全ての方に共有したい話です。

たつきさんに相談してみたい方へ

「AIに興味はあるけれど、何から触ればいいか分からない」
「AIで何か作ってみたけれど、誰かの役に立つのか自信がない」
「自分のキャリアを、AIと一緒に育てるってどういうこと？」

そんな問いを持っている方にとって、たつきさんはちょうど一歩手前にいる伴走者だと思います。完璧な専門家ではなく、自分のドラえもんを自分で作っている人だからこそ、相談しやすい距離感があるはずです。

たつきさんの「AIキャリア伴走者ドラえもん」、いつか公開されたり、その実例が見られる日を、私も楽しみにしています。

最後まで読んでくださった方、ありがとうございました。

田畑さなえ｜note運用代行・取材ライター

このサンプルと一緒に、スキルが出力したもの

  • MTG要約：本人語を7箇所そのまま引用＋触れない方がよい話＋要確認情報5項目
  • 記事ネタ候補：3本（推奨度★3／★2／★1）
  • 章立て：採用案の10章構成
  • 投稿前チェック：8カテゴリ判定（事実確認・NGテーマ・トーン・売り込み感 etc）
  • クライアント確認事項：12項目（プロダクション名の掲載可否・第三者のお名前掲載可否・GPTs公開URL等）

📂 受け取り方

スキル本体（sanae-note-opsフォルダ）を、さなえさんのMacの ~/.claude/skills/ 配下に置くだけで使えるようになります。

手順

  1. こちらからフォルダをダウンロード（おけもんから個別にお渡しします）
  2. 解凍して sanae-note-ops フォルダを ~/.claude/skills/ に配置
  3. Claude Codeを開いて「sanae-note-ops 起動して」と話しかける
💬 セットアップ伴走OK： もし「自分のMacにコピーするやり方わかんない」だったら、画面共有で5分くらいで一緒にやれるのでいつでも声かけてください。

🚶 最初の使い方の提案

いきなり本番クライアントで使うのが不安だったら、こんな順で慣らすのがおすすめです：

  1. 担当クライアント1社を選んで、カルテだけ埋める（30〜60分・空欄でもOK）
  2. 直近のMTG文字起こしを1本だけ貼って、MTG要約と記事ネタ3〜5本まで 出してもらう
  3. ピンと来た1本を選んで、叩き台まで作ってもらう
  4. 出てきた叩き台を読んで「ここ違和感ある」「ここはもっとこう」をフィードバック
  5. そのフィードバックをスキル本体に反映してさなえさん専用としてどんどん育てる

さなえさんへのお願い

このスキル、さなえさんが使えば使うほど精度が上がる設計になっています。「叩き台のここが違和感あった」「もっとこう書いてほしい」みたいなフィードバックを教えてもらえると、さなえさん専用としてもっと磨けるので、ぜひ遠慮なく言ってください。

押し付けるつもりはぜんぜんなくて、「楽しくてやってる延長で作ったから、もし合いそうだったら触ってみて」 くらいの温度で受け取ってもらえたら嬉しいです🌱
