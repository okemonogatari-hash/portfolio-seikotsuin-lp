既に持ってる仕組み × 機能しなかった理由

持ってる仕組み機能しなかった理由
Intent Capsule v0.1（毎ターン頭注入）載っているが、carry_over を 読んだだけで動かなかった
Karpathy 4原則（kernel invariant）Think Before Coding を実践せず、受けた瞬間に動いた
anti-silo-check.sh（毎ターン物理注入）中継表現の検出には効いたが、「自分1人で受けるな」検出は未実装
過去実績確認ルールhook 通知は出ているが、私たちが 実際に読みに行く 動作を怠った

🛠 物理化すべき次の対策

  1. 「Zoom案件キーワード検知」hook：会話に「Zoom」「録画」「トランスクリプト」が出たら、自動で PJ_Zoom文字起こし自動Docs化/STATE.md と 【マスター】Zoom API 接続・運用ガイド を強制Read
  2. 「事案#XX」hook：直近30日の PDCA成長記録/事案サンプル_*.md のタイトル一覧を毎セッション開始時に強制提示
  3. 「自分のPJ呼び」検知：はるかが受けた話題が他CxOのPJ owner と一致したら、即その CxO を召喚する強制プロトコル
  4. 正本マスターレジストリ：【マスター】*.md 一覧をルート直下に index 化し、セッション開始時に強制提示
